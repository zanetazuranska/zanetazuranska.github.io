Mario made by Zengine

Zengine is a custom-built 2D platformer engine developed in C++ using SFML, created to explore low-level game architecture and systems design. The project was designed as a modular and scalable foundation for 2D games, with a strong focus on performance, flexibility, and clean structure.

To demonstrate the engine’s capabilities, I developed a playable Mario-inspired prototype. This version reimagines classic platforming mechanics with my own approach to gameplay systems and implementation.

Project Context

The project was created as part of the Girls in Game initiative, in collaboration with CD PROJEKT RED and under the mentorship of Przemysław Machniewski. It was an opportunity to work with professional development practices while building a fully custom engine from scratch.

Engine Development (Zengine)

I was responsible for designing and implementing the core architecture and systems of the engine, including:

  • Input handling module
  • Rendering system (SFML-based)
  • Audio system
  • Multiple map support
  • Physics system
  • Collision detection and resolution
  • User Interface framework
  • Animation system
  • Debugging tools
  • Custom data structures

Mario Prototype

The Mario prototype was built entirely on top of Zengine to validate the engine’s capabilities in a real gameplay scenario. It includes core platformer mechanics such as movement, jumping, collisions, and interactions, all implemented using my custom systems.

Rather than recreating the original game, the goal was to reinterpret it and demonstrate how the engine can support complete gameplay experiences.
